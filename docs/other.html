12. Other¶

EvenTDT offers additional functionality to help you create TDT applications.

Logger¶

A logging interface.

class LogLevel(value)¶

The logger’s logging level. It is based on a OrderedEnum.

Valid logging levels:

  1. INFO - Information and higher-level logging only

  2. WARNING - Warnings and higher-level logging only

  3. ERROR - Errors and higher-level logging only

log_time()¶

Get the time string.

Returns

The time string.

Return type

str

process_name(process)¶

Get the string with the name of the process.

Parameters

process (str) – The name of the process, if available.

Returns

A string with the formatted name of the process. If no process name is given, this function returns an empty string.

Return type

str

set_logging_level(level)¶

Set the logging level.

Parameters

level (LogLevel) – The logging level.

info(*args, process=None)¶

Log an information message. All arguments are passed on to the print function.

Parameters

process (None or str) – The name of the process calling the log. If it is given, the logger prints part of its name while logging.

warning(*args, process=None)¶

Log a warning. All arguments are passed on to the print function.

Parameters

process (None or str) – The name of the process calling the log. If it is given, the logger prints part of its name while logging.

error(*args, process=None)¶

Log an error. All arguments are passed on to the print function.

Parameters

process (None or str) – The name of the process calling the log. If it is given, the logger prints part of its name while logging.

Base Classes¶

EvenTDT includes some base classes that add to the functionality of some classes or which require implementations.

Attributable objects are normal objects that can have attributes.

class Attributable(attributes=None)¶

Attributable objects have a dictionary that stores attributes.

Variables

attributes (dict) – The object’s attributes. The keys are the attribute names and the values are the respective values.

__init__(attributes=None)¶

Initialize the object with the attributes.

Parameters

attributes (dict or None) – The starting attributes. If None is given, an empty dictionary is created instead.

__weakref__¶

list of weak references to the object (if defined)

Exportable objects are normal objects that can be exported as a JSON string and loaded back.

class Exportable¶

An abstract class of an object that can be exported as a JSON string and imported back.

abstract to_array()¶

Export the object as a dictionary.

Returns

The object as a dictionary.

Return type

dict

abstract static from_array(array)¶

Create an instance of the object from the given dictionary.

Parameters

array (dict) – The dictionary with the attributes to create the object instance.

Returns

A new instance of an object with the same attributes stored in the object.

Return type

object

static encode(data)¶

Try to encode the given data. This function expects a dictionary, a list or an object and checks if values are JSON serializable. If this is not possible, instances of Exportable are converted to arrays. This is done through the to_array() function.

Parameters

data (dict or list) – The data to encode.

Returns

The encoded data.

Return type

dict or list or object

static decode(data)¶

A function that recursively decodes cached data. By decoded, it means that objects are created where necessary or possible. Only classes that inherit the Exportable can be decoded. This is done through the from_array() function.

Note

When decoding, the function expects either a dictionary or a list. JSON objects cannot be anything else.

Parameters

data (dict or list) – The data to decode.

Returns

A dictionary, list or object, but this time decoded.

Return type

dict or list or object
