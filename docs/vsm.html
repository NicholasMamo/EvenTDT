2. Vector Space Model (VSM)¶

Most document representation occurs in the Vector Space Model (VSM). EvenTDT provides classes to help you work with the VSM. Keep reading to learn more about the different VSM classes available in EvenTDT.

Vectors¶

The Vector is the basis of the Vector Space Model (VSM). The class contains is used in clustering and Natural Language Processing (NLP) to represent documents.

class Vector(dimensions=None, *args, **kwargs)¶

The Vector class is the smallest building block in the Vector Space Model (VSM). It is used for tasks such as clustering and to represent documents. Vectors are based on Attributable so that they may have additional properties.

Vector Math¶

Mathematical functions related to vectors.

magnitude(v)¶

Get the magnitude of the given vector. The magnitude is computed as:

\[||v|| = \sqrt{\sum_{n=1}^{V} {v_n^2}}\]

where \(v\) is a vector having \(V\) dimensions.

Parameters

v (Vector) – The vector whose magnitude will be calculated.

normalize(v)¶

Normalize the given vector. Normalization is computed as:

\[f = \frac{f}{||v||}\]

where \(f\) is a feature in vector \(v\).

Parameters

v (Vector) – The vector that will be normalized.

augmented_normalize(v, a=0.5)¶

Normalize the given vector using the formula:

\[f = a + (1 - a) \frac{f}{x}\]

where \(x\) is the magnitude of the highest dimension \(f\) in the vector. \(a\) is the augmentation, between 0 and 1, inclusive.

Parameters

  • v (Vector) – The vector that will be normalized

  • a (float) – The minimum magnitude of each dimension.

Raises

ValueError – When the augmentation is not between 0 and 1

concatenate(vectors)¶

Concatenate a list of vectors and return a new vector. This means adding the dimensions together:

\[f = \sum_{i=1}^{|V|}{f_i}\]

where \(f\) is the weight of the concatenated vector, and \(f_i\) is the weight of the same feature f in each vector in the set \(V\).

Parameters

vectors (list of Vector instances) – A list of vectors

Returns

A single vector

Return type

Vector

euclidean(v1, v2)¶

Compute similarity using Euclidean distance. The Euclidean distance \(e_{p, q}\) is computed as:

\[e_{p, q} = \sqrt{ \sum_{i=1}^{n}{ (q_i - p_i)^2 } }\]

Where \(q_i\) is feature \(i\) in vector \(q\), and \(p_i\) is the same feature \(i\) in vector \(p\). \(n\) is the union of features in vectors \(q\) and \(p\).

Parameters

  • v1 (Vector) – The first vector.

  • v2 (Vector) – The second vector.

manhattan(v1, v2)¶

Compute similarity using Manhattan distance. The Manhattan distance \(m_{p, q}\) is computed as:

\[m_{p, q} = \sum_{i=1}^{n}{ |q_i - p_i| }\]

Where \(q_i\) is feature \(i\) in vector \(q\), and \(p_i\) is the same feature \(i\) in vector \(p\). \(n\) is the union of features in vectors \(q\) and \(p\).

Parameters

  • v1 (Vector) – The first vector.

  • v2 (Vector) – The second vector.

cosine(v1, v2)¶

Compute similarity using cosine similarity. The cosine similarity \(cos_{p, q}\) is computed as:

\[cos_{p, q} = \frac{\sum_{i=1}^{n}{ q_i \cdot p_i }}{ ||p|| + ||q|| }\]

Where \(q_i\) is feature \(i\) in vector \(q\), and \(p_i\) is the same feature \(i\) in vector \(p\). \(n\) is the intersection of features in vectors \(q\) and \(p\).

Parameters

  • v1 (Vector) – The first vector.

  • v2 (Vector) – The second vector.

cosine_distance(v1, v2)¶

Compute the cosine distance. The cosine distance \(cosd_{p, q}\) is computed as:

\[cosd_{p, q} = 1 - cos_{p, q}\]

Warning

The cosine distance is not a real distance metric.

Parameters

  • v1 (Vector) – The first vector.

  • v2 (Vector) – The second vector.

Clustering¶

Clusters are groups of Vector instances. Groups are meant to maximize distance between vectors of different subjects. Simultaneously, they aim to minimize the distance between vectors in the same clusters.

Clusters can be created normally, or they can be generated by a ClusteringAlgorithm.

Algorithms¶

Clusters are normally the product of clustering algorithms A clustering algorithm takes in vectors or documents and clusters them.

Clustering algorithms are represented as classes so that they maintain their state. Each class stores, at least, a list of Cluster instances. New documents can be clustered by calling the cluster() method.

class ClusteringAlgorithm¶

Clustering algorithms maintain a state. The state is, at least, a list of vsm.clustering.cluster.Cluster instances.

The No-K-Means algorithm is an incremental clustering algorithm, described in research by Azzopardi et al. in 2016. The algorithm compares incoming documents to existing clusters. The vector is added to the most similar cluster if the similarity exceeds a certain threshold. Otherwise, the vector is added to a new cluster.

Since clusters accumulate over time, the algorithm also has a freeze period. Clusters that have not been updated with new vectors for an equivalent period are frozen. No new vectors can be added to frozen clusters.

class NoKMeans¶

The No-K-Means algorithm is an incremental clustering algorithm. Incoming vectors are added to the most similar cluster if the similarity exceeds a threshold. Otherwise, a new cluster is created just for them.

This class contains one additional field - the frozen clusters. Inactive clusters are frozen out, and are not checked with incoming vectors anymore.

Variables

_frozen_clusters (list of Cluster instances) – A list of clusters that have been retired.

class TemporalNoKMeans¶

The Temporal No-K-Means algorithm is an incremental clustering algorithm. In contrast with the normal No-K-Means, its freeze period is in seconds, not on the number of vectors seen so far. It is assumed that the vectors are provided in chronological order across function calls. Within function calls, vectors are sorted.
